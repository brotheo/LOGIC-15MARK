TEACHER GUIDE

15-Mark Question – Random Number Fairness

Lesson Details

Duration: 60–75 minutes

Objectives: Identify array structures, generate/store random numbers, count frequency, sort a 2D array, and output probabilities to 4 d.p.

Lesson Delivery Plan
PART 1: Starter (10 mins)

Ask: If I generate 100 numbers between 1 and 10, how do I check if it is fair?

Expected ideas: Count how many times each number appears, compare frequencies.

"Today we scale this to 100,000 numbers like an exam question."

PART 2: Decode the Question (15 mins)

Step 1: Identify Data Structures

Go through the array structures with students (see Answer Key below).

Step 2: Identify Question Type

Ask: What type of 15-mark question is this?

Answer: Simulation, Frequency, Sorting.

Step 3: Break into Tasks

Generate numbers → Store numbers → Count frequency → Sort → Calculate chance → Output.

PART 3: Teacher Modelling (20 mins)

"We prepare storage before counting"

FOR X ← 1 TO 10 CountedNumber[X,1] ← X CountedNumber[X,2] ← 0 NEXT X

KEY SKILL: Generate + Count

"We combine generation + counting in ONE loop"

Temp ← ROUND(RANDOM() * 9, 0) + 1 CountedNumber[Temp,2] ← CountedNumber[Temp,2] + 1

Sorting Concept: We are sorting Frequency (column 2). Emphasise that we must swap BOTH columns.

Chance Formula: chance = frequency / 100000

PARTS 4-6: Practice & Plenary (30 mins)

Guided Practice (15m): Students complete worksheet. Walk around and support.

Independent Practice (10m): Students attempt full pseudocode on paper.

Plenary (5m): Discuss why we initialise frequency, why swap two columns, and why round to 4 d.p.

WORKSHEET ANSWER KEY

Section A: Data Structures

RandomNumber[] stores all generated numbers

CountedNumber[X,1] stores number (1-10)

CountedNumber[X,2] stores frequency

Section B: Initialisation

CountedNumber[X,1] ← X

CountedNumber[X,2] ← 0

Section C: Generation & Counting

ROUND(RANDOM() * 9, 0) + 1

RandomNumber[X] ← Temp

CountedNumber[Temp,2] ← CountedNumber[Temp,2] + 1

Section D: Sorting Condition

IF CountedNumber[I,2] < CountedNumber[I+1,2]

Why? Because we are sorting in descending order (highest frequency first).

Section E: Swap Logic
CountedNumber[I,1] ← CountedNumber[I+1,1] CountedNumber[I,2] ← CountedNumber[I+1,2] CountedNumber[I+1,1] ← Temp1 CountedNumber[I+1,2] ← Temp2
Section F: Output

OUTPUT "Number: ", CountedNumber[I,1]

OUTPUT "Chance: ", ROUND(CountedNumber[I,2] / 100000, 4)

TEACHER NOTES & STRATEGIES

Scaffolding for Weaker Students

Do not expect them to complete the sorting algorithm independently. Focus heavily on:

  • Array initialisation
  • Random number generation
  • Counting the frequency
Pushing Stronger Students

Expect a full, perfectly indented solution from start to finish. Focus on:

  • The full Bubble Sort algorithm including the swap logic
  • Correct output formatting
Critical Teaching Insight

Students traditionally struggle most with:

  • 2D array meaning: Keep reinforcing what Column 1 and Column 2 represent.
  • Swapping correctly: They often forget they must swap both the number and its frequency to keep them linked.
  • Combining loops: They may try to loop 100,000 times for everything instead of isolating the counting.

👉 Spend extra time checking for understanding on these points.
