LESSON PLAN

Topic: 15-Mark Question – Random Number Fairness
Learning Objectives By the end of the lesson, students will be able to:
  • Identify array structures in a scenario
  • Generate and store random numbers (1–10)
  • Count frequency using arrays
  • Sort a 2D array (descending)
  • Calculate and output probabilities (4 d.p.)
Lesson Duration

60–75 minutes

PART 1: Starter (10 mins)

Quick Question (Think–Pair–Share)

Ask:

If I generate 100 numbers between 1 and 10, how do I check if it is fair?

Expected ideas:

  • Count how many times each number appears
  • Compare frequencies
Lead into:

"Today we scale this to 100,000 numbers like an exam question."

PART 2: Decode the Question (15 mins)

Step 1: Identify Data Structures

Ask students to fill:

Structure Meaning
RandomNumber[]
CountedNumber[row,1]
CountedNumber[row,2]
Answers:
RandomNumber[] → stores all generated numbers
Column 1 → number (1–10)
Column 2 → frequency

Step 2: Identify Question Type

Ask:

"What type of 15-mark question is this?"

Answer: Simulation, Frequency, Sorting

Step 3: Break into Tasks

Students list:

  • Generate numbers
  • Store numbers
  • Count frequency
  • Sort
  • Calculate chance
  • Output
PART 3: Teacher Modelling (20 mins)

Step-by-Step Build

Step A: Initialise Array

FOR X ← 1 TO 10 CountedNumber[X,1] ← X CountedNumber[X,2] ← 0 NEXT X

Explain: "We prepare storage before counting"

Step B: Generate + Count (KEY SKILL)

Temp ← ROUND(RANDOM() * 9, 0) + 1 CountedNumber[Temp,2] ← CountedNumber[Temp,2] + 1
Emphasise: "We combine generation + counting in ONE loop"

Step C: Sorting (Concept First)

Ask: "What are we sorting?"

Answer: Frequency (column 2)

Important: "We must swap BOTH columns"

Step D: Output with Formula

Chance formula:

chance = frequency / 100000
PART 4: Guided Practice (15 mins)

Students complete partially written code (see worksheet below).

Teacher walks around and supports.

PART 5: Independent Practice (10–15 mins)

Students attempt full pseudocode.

PART 6: Plenary (5 mins)
Ask:
  • Why must we initialise frequency?
  • Why do we swap two columns?
  • Why round to 4 d.p.?

STUDENT WORKSHEET (SCAFFOLDED)

Section A: Understanding the Problem

Fill in the blanks:

  • The array RandomNumber[] stores
  • The array CountedNumber[X,1] stores
  • The array CountedNumber[X,2] stores
Section B: Complete the Code (Easy)

Initialise the array

FOR X ← 1 TO 10 CountedNumber[X,1] ← ______ CountedNumber[X,2] ← ______ NEXT X
Section C: Random Number Generation

Fill the missing parts:

FOR X ← 1 TO 100000 Temp ← ROUND(RANDOM() * ___, 0) + ___ RandomNumber[X] ← ______ CountedNumber[____,2] ← CountedNumber[____,2] + 1 NEXT X
Section D: Sorting (Medium)

Complete the condition:

IF CountedNumber[I,2] ___ CountedNumber[I+1,2]

Should it be: <

Why?

Section E: Swap Logic (Challenge)

Fill in:

Temp1 ← CountedNumber[I,1] Temp2 ← CountedNumber[I,2] CountedNumber[I,1] ← __________________ CountedNumber[I,2] ← __________________ CountedNumber[I+1,1] ← ________________ CountedNumber[I+1,2] ← ________________
Section F: Output (Easy)

Complete:

OUTPUT "Number: ", __________ OUTPUT "Chance: ", ROUND(__________ / 100000, 4)
Section G: Full Challenge (Exam Style)

Write the full pseudocode using:

  • Correct declarations
  • Loops
  • Sorting
  • Output
EXTENSION (Top Students)
Ask:
  • How would you check if the generator is fair?
  • What if numbers were 1–20 instead of 1–10?

TEACHER NOTES (IMPORTANT)

Differentiation

Weak students:

Focus only on:

  • Initialise
  • Generate
  • Count
Strong students:

Full solution including sorting + output

Key Teaching Insight

Students struggle most with:
  • 2D array meaning
  • Swapping correctly
  • Combining loops

Spend extra time here.
